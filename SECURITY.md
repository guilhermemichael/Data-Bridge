# Security Policy

## Supported Versions

This project is under active development. Security reports should target the latest version available on the `main` branch.

## Reporting a Vulnerability

If you find a vulnerability, do not open a public issue with exploit details.

Contact:

- GitHub: @guilhermemichael
- Email: guilherme.michael.dev@gmail.com

## Security Scope

Relevant areas include:

- Authentication
- Authorization
- File upload validation
- Path traversal prevention
- Environment variable exposure
- SQL injection prevention
- Token handling
- Sensitive logging
